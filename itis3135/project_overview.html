Requirements & Design Proposal: Photography Portfolio

Developer: Sean Briganti | Date: April 5, 2026

Project Overview

  • Application Description: A professional, read-only portfolio website for a freelance photographer. The site will establish an online presence, showcase their work with a distinct cinematic and gritty aesthetic, and provide a streamlined way for potential clients to view rates and get in touch.
  • Intended Users: The general public, photography lovers, and potential clients looking to book photoshoots.
  • Content Overview: High-resolution photography galleries, professional biography, service packages/pricing, and contact information.

Page Design Details

1. Home Page (index.html)

  • Purpose: Landing page to immediately capture attention with the photographer's best cinematic shot.
  • Audience: General public, potential clients.
  • Content: Hero image banner, brief welcome text, call-to-action button linking to the portfolio.
  • Data Entry: None.
  • Actions: Button click navigates to `portfolio.html`.

2. Portfolio Gallery (portfolio.html)

  • Purpose: To showcase the photographer's body of work in high resolution.
  • Audience: Photography lovers, potential clients.
  • Content: Grid of thumbnail images.
  • Data Entry: None.
  • Actions: Clicking a thumbnail opens an interactive JavaScript lightbox/carousel to view the image full-screen.

3. Services & Rates (services.html)

  • Purpose: Provide details on photoshoot packages and pricing.
  • Audience: Potential clients.
  • Content: Text descriptions of packages (e.g., Portrait, Event, Street Photography).
  • Data Entry: None.
  • Actions: Uses a jQuery-UI Accordion. Users click on a package header to expand and reveal the pricing details below it.

4. About the Artist (about.html)

  • Purpose: Background information on the photographer and their artistic philosophy.
  • Audience: General public.
  • Content: Portrait of the photographer, biography text, and a section for client testimonials.
  • Data Entry: None.
  • Actions: Page load triggers an AJAX call to load testimonial data from a local JSON file into the DOM.

5. Contact & Booking (contact.html)

  • Purpose: Allow users to inquire about booking a shoot.
  • Audience: Potential clients.
  • Content: Contact form and social media links.
  • Data Entry: Yes. Form fields: Name, Email, Type of Shoot (Dropdown), Date, Message.
  • Validation: JavaScript validation to ensure Email is formatted correctly and Name/Message are not blank before simulating a submission.
  • Actions: Clicking "Submit" triggers validation. If successful, displays a "Message Sent" alert (read-only mockup, no backend database routing).

Dynamic Functionality

  • JavaScript Interactivity (Lightbox Gallery): Located on `portfolio.html`. When a user clicks a photo thumbnail, a dark, immersive overlay appears showing the full-size image with "Next" and "Previous" navigation. This enhances the gritty, cinematic viewing experience. Inspiration: https://fancyapps.com/fancybox/
  • jQuery-UI Widget (Accordion): Located on `services.html`. To prevent overwhelming the user with text, the photography packages will be hidden inside an accordion. Clicking a header expands the text. Inspiration: https://jqueryui.com/accordion/
  • AJAX Functionality (Testimonial Loader): Located on `about.html`. I will use the JavaScript `fetch()` API (AJAX) to request a local `reviews.json` file containing static client feedback. This data will be dynamically injected into the HTML without reloading the page, fulfilling the AJAX requirement while adhering to the read-only (No CRUD) constraint.
